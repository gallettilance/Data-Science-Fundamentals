Kmeans ++¶

a) What is the difference between K means and K means ++?

The difference between Kmeans and Kmeans ++ is the initialization process. While with Kmeans, we randomly pick k centers. On the other hand, we pick our initial center randomly, but Kmeans ++ picks the next centers proportional to the distances of points to the centers picked so far. Kmeans++ helps with the case when there exists the initialization of two centroids really close to each other, which means a higher chance to split up a cluster

b) What are some limitations of K means ++?

K means ++ can still to choose outliers as centroids (of their own clusters) and kmeans ++ prefers globular structures so that any non-globular dataset are hard to work with.

Input:¶

Task:¶

  • Implement a hierarchical clustering algorithm using Hamming distance as the metric clustering DNA sequences.

Definition of Hamming Distance:¶

The Hamming distance between two strings of equal length is the number of positions at which the corresponding symbols are different. Mathematically, if we have two strings, $s$ and $t$, of equal length, then the Hamming distance $H(s, t)$ is given by:

$$ H(s, t) = \sum_{i=1}^{n} [s_i \neq t_i] $$

where $n$ is the length of the strings, $s_i$ and $t_i$ are the characters at position $i$ in $s$ and $t$ respectively, and $[s_i \neq t_i]$ is an indicator function, equal to 1 if $s_i \neq t_i$ and 0 otherwise.

def hamming_dist(s, t):
    dist = 0
    for i in range(len(s)): #since equal length strings
        if s[i] != t[i]: #count to distance if corresponding symbols are different
            dist += 1
    return dist

# hamming_dist(s1, s2)

def get_distance_matrix(sequences):
    len_seq = len(sequences)
    dist_matrix = np.zeros((len_seq, len_seq)) #initialize to empty n x n array of the length of the sequence

    for i in range(len_seq):
        for j in range(i+1, len_seq):
            dist_matrix[i][j] = hamming_dist(sequences[i], sequences[j]) #calculate the hamming distance for each data point in the matrix
            dist_matrix[j][i] = dist_matrix[i][j] #mirror

    return dist_matrix

#get_distance_matrix(sequences)
def hierachical_clustering_singlelink(sequences):
    distance_matrix = get_distance_matrix(sequences)
    linkage_matrix = linkage(distance_matrix, method="single") #Performs hierarchical/agglomerative clustering on the (hamming) distance matrix 
    dendrogram(linkage_matrix) #use scipy's dendrogram function to illustrate how eaxch cluster is composed
    plt.show()
